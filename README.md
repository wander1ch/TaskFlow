# TaskFlow

Коллаборативная система управления задачами, построенная на языке Go.

## 1. Функционал

TaskFlow предоставляет богатый набор возможностей для совместной работы над проектами и задачами:

- **Регистрация и аутентификация пользователей:**
  - Безопасная регистрация пользователей с шифрованием паролей с помощью Bcrypt.
  - Авторизация на основе JWT-токенов (JSON Web Tokens) с поддержкой сессий.

- **Управление командами (Teams):**
  - Возможность создавать команды для совместной работы.
  - Добавление и управление участниками команд.
  - Ролевая модель доступа внутри команд (RBAC):
    - **Owner** (Владелец) — полный контроль над командой и её задачами.
    - **Manager** (Менеджер) — создание, распределение и изменение задач.
    - **Member** (Участник) — выполнение назначенных задач и обновление их статуса.

- **Управление задачами (Tasks):**
  - Полный цикл CRUD-операций (Создание, Чтение, Обновление, Удаление) для задач внутри команд.
  - Назначение исполнителей на конкретные задачи.
  - Гибкое отслеживание статусов выполнения задач.

- **История изменений и аудит (Task History & Audit Log):**
  - Логирование действий пользователей с задачами.
  - Возможность отследить историю изменений параметров задач для прозрачности рабочего процесса.

- **Аналитика и статистика:**
  - Сбор метрик и аналитики по эффективности работы команды, количеству выполненных задач и текущей загрузке.

- **Высокопроизводительное кэширование:**
  - Интеграция с Redis для кэширования часто запрашиваемых данных, что снижает нагрузку на основную базу данных и ускоряет отклики API.

- **Веб-интерфейс (Frontend):**
  - Простой и отзывчивый интерфейс пользователя (Single Page Application на базе HTML/CSS/JS в папке `web/static`), позволяющий наглядно взаимодействовать с бэкендом.

## 2. Способ использования

### Требования
Для успешного запуска проекта вам потребуются:
- **Docker** и **Docker Compose**
- **Go 1.25** или новее (только для локальной разработки без Docker)

---

### Быстрый запуск (Рекомендуемый способ через Docker)

Все необходимые зависимости (PostgreSQL, Redis, миграции и само приложение) упакованы в контейнеры и запускаются одной командой.

1. **Клонируйте репозиторий** и перейдите в директорию проекта.
2. **Запустите контейнеры**:
   ```bash
   make docker-up
   ```
   *Эта команда соберёт приложение, накатит все SQL-миграции в PostgreSQL и запустит сервис на порту `8080`.*

3. **Откройте приложение**:
   Перейдите в браузере по адресу: **`http://localhost:8080`**

4. **Остановите контейнеры** (при необходимости):
   ```bash
   make docker-down
   ```

---

### Локальная разработка и тестирование

Если вы хотите запускать и изменять код локально:

- **Запуск тестов:**
  Выполните команду для запуска всех юнит- и интеграционных тестов:
  ```bash
  make test
  ```

- **Запуск бэкенда локально:**
  Для этого у вас должны быть запущены PostgreSQL и Redis локально. Настройте параметры подключения в файле `.env`.
  ```bash
  make run
  ```

- **Управление миграциями базы данных:**
  Применить новые миграции:
  ```bash
  make migrate-up
  ```
  Откатить миграции:
  ```bash
  make migrate-down
  ```

- **Проверка кода линтером:**
  ```bash
  make lint
  ```

- **Генерация документации API (Swagger):**
  ```bash
  make swagger
  ```

### Документация API (Swagger)
После запуска приложения вы можете ознакомиться с интерактивной документацией API, сгенерированной с помощью Swagger.

1. Запустите приложение.
2. Перейдите в браузере по адресу: **`http://localhost:8080/swagger/index.html`**

## 3. Что было использовано в проекте

Проект разработан в соответствии с современными стандартами разработки на Go, используя чистую архитектуру (Clean Architecture) и контейнеризацию.

### Технологический стек бэкенда:
- **Go (Golang) v1.25** — основной компилируемый язык программирования.
- **Gin Web Framework (`github.com/gin-gonic/gin`)** — быстрый и гибкий HTTP-фреймворк для реализации RESTful API и раздачи статических файлов.
- **JWT-Go (`github.com/golang-jwt/jwt/v5`)** — библиотека для работы с JSON Web Tokens, обеспечивающая безопасность сессий пользователей.
- **PGX (`github.com/jackc/pgx/v5`)** — современный, производительный драйвер PostgreSQL для Go с поддержкой пула соединений.
- **Go-Redis (`github.com/redis/go-redis/v9`)** — эффективный клиент для кэширования данных в Redis.
- **Google UUID (`github.com/google/uuid`)** — генератор уникальных идентификаторов для сущностей (задач, пользователей, команд).
- **Bcrypt (`golang.org/x/crypto/bcrypt`)** — криптографическое хэширование паролей для защиты пользовательских данных.
- **Testify (`github.com/stretchr/testify`)** — фреймворк для написания чистых и выразительных тестов с утверждениями (asserts) и моками.

### СУБД и инфраструктура:
- **PostgreSQL 15 (Alpine-образ)** — основная реляционная СУБД для надежного хранения структурированных данных.
- **Redis 7 (Alpine-образ)** — in-memory хранилище данных, используемое в качестве кэша.
- **golang-migrate (`migrate/migrate`)** — инструмент версионирования схемы базы данных, автоматически применяющий SQL-миграции при старте приложения.
- **Docker и Docker Compose** — для изоляции окружения и бесшовного развёртывания проекта одной командой на любой ОС.

### Архитектурные подходы:
- **Clean Architecture (Чистая архитектура):**
  Код проекта разделён на независимые слои:
  - `domain` — ядро бизнес-логики (модели данных).
  - `repository` — слой прямого взаимодействия с базой данных (Postgres).
  - `service` — слой бизнес-правил, координирующий работу репозиториев и кэша.
  - `transport` — HTTP-хендлеры, валидация входящих запросов и формирование ответов.
- **Кэширование "Cache-Aside":** Приложение сначала пытается прочитать данные из Redis, а в случае промаха обращается к PostgreSQL и наполняет кэш.
- **Принцип слабой связанности (Dependency Injection):** Зависимости явно передаются через конструкторы, что упрощает юнит-тестирование с использованием мок-объектов.
